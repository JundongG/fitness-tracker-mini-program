# 健身跟踪管理小程序 - 初步设计 (v0.1)

## 技术栈建议
- **平台**：微信小程序（最符合“小程序”需求，生态成熟，用户基数大）
- **前端**：原生小程序 + WXML/WXSS/JS
- **数据存储**：
  - 首选：微信云开发 (云数据库 + 云存储 + 云函数)
  - 备选：本地 storage + 自定义后端 (Node/Express or 其他)
- **图表**：echarts-for-wechat 或 wxcharts
- **状态管理**：小程序原生 + 简单全局 data
- **部署**：微信开发者工具 + 云开发

## 数据模型 (初步)
- User: { openid, nickname, goals: [...] }
- Workout: { id, date, exercises: [{name, sets: [{weight, reps, rest}], notes}] }
- Exercise: { name, category (chest/leg/etc), description, video? }
- Progress: { date, weight, bodyFat?, measurements? }

## UI 流程 (核心)
1. 首页/Tab: 快速记录训练
2. 选择或搜索动作 → 添加组数 (重量/次数/时间)
3. 保存训练 → 自动更新历史和统计
4. 历史页：列表 + 按动作筛选 + 详情
5. 我的页：目标、图表、个人记录 (PR)

## MVP 范围 (建议)
- 基础训练记录 + 保存
- 历史查看 (最近10条)
- 简单统计 (本周次数、总重量趋势)
- 预置动作库 (50+ 常见动作)
- 本地优先 + 云同步 (云开发)

## 风险与注意
- 小程序审核：避免医疗/健康夸大宣传
- 数据隐私：用户健身数据敏感，做好本地加密或合规
- 离线体验：必须支持无网记录

- 完整数据字典
