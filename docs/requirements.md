# 健身跟踪管理小程序 - 需求规格 (v0.1)

## 1. 项目背景
用户希望有一个简单、专注的工具来跟踪健身进度，记录训练，查看进步，避免使用复杂App的负担。

## 2. 目标用户
- 健身新手和中级爱好者
- 每周训练3-6次
- 希望数据可视化但操作简单
- 主要在手机上使用

## 3. 核心功能 (MVP优先级排序)

### P0 - 必须有 (MVP)
- **训练记录**：快速添加训练（动作、组数、重量/时长、备注）
- **历史查看**：按日期、按动作查看历史记录
- **简单仪表盘**：本周训练次数、总时长、个人纪录 (PR)
- **动作库**：预设常见健身动作（可自定义添加）

### P1 - 重要
- **目标设定**：设定每周训练目标、重量目标
- **进度图表**：简单折线图/柱状图显示进步（重量、次数、体重趋势）
- **训练计划**：基础模板（推/拉/腿、全身等），可选择计划开始训练
- **提醒**：训练日提醒

### P2 - 锦上添花 (后续迭代)
- 营养/饮食简单记录
- 照片进度对比
- 导出数据 (PDF/Excel)
- 社交分享单次训练
- 成就系统/徽章

## 4. 非功能需求
- 离线可用（小程序特性）
- 数据本地+云同步（如果用云开发）
- 响应快，界面干净
- 支持中文为主

## 5. 技术考虑 (待确认)
- 平台：微信小程序（推荐）
- 存储：微信云开发 (云数据库 + 云存储) 或 本地 + 自定义后端 (Node/Express or 其他)
- 图表：wx-charts 或 echarts-for-wechat
- 状态管理：小程序原生 或 轻量框架

## 6. 待澄清问题
- 是否需要用户账号系统？（微信登录即可？）
- 是否支持多用户家庭/教练查看？
- 体重/体脂等身体数据跟踪要不要？
- 有没有特定训练类型偏好（力量、跑步、瑜伽、CrossFit等）？
- 预算/时间线？
- 是否已有设计稿或参考App？

## 7. 成功指标 (MVP后)
- 用户能在一分钟内完成一次训练记录
- 连续使用7天以上用户比例
- 正面反馈：简单好用
